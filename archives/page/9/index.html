原码,反码,补码和移码

原码的定义：

①小数原码的定义 
















 

[X]原 =

 X  0≤X ＜1
 1－ X －1 ＜ X ≤ 0

 例如： X=+0.1011 , [X]原= 01011
             X=－0.1011  [X]原= 11011

②整数原码的定义
















 

[X]原 =

 X  0≤X ＜2n
 2n－X －2n ＜ X ≤ 0

 例如： X=+1011 , [X]原= 01011
            X=－1011  [X]原= 11011

原码和反码，补码都是一样的，都是正数本身。

原码的符号位正为0、负为1。原码的加减计算不能将符号位一起纳入加减计算。例如：001(+1)+101(-1)=110（-2）。

很显然结果应该是0。为了解决这种问题，就出现了反码。

反码的定义：

①小数反码的定义 
















 

[X]反 =

 X    0≤X ＜1
 2－2n-1－X   －1 ＜ X ≤ 0

例如：   X=+0.1011      [X]反= 01011
         X=－0.1011     [X]反= 10100

②整数反码的定义
















 

[X]反 =

 X    0≤X ＜2n
 2n+1－1－X    － 2n ＜ X ≤ 0

例如：   X=+1011      [X]反= 01011
         X=－1011     [X]反= 10100

反码是符号位为1,其它位是原码取反。

反码解决了将符号位纳入加减计算的功能，例如：001(+1)+110(-1)=111（0）。

如果空间的大小是3位的，那么反码表示的范围“-3到-0”和“+0到3”。可见-0与+0是多余了一个0。为了解决这个问题即出现了补码。

补码的定义：

①小数补码的定义
















 

[X]补 =

 X   0≤X ＜1
 2＋ X   －1 ≤ X ＜ 0

例如：  X=+0.1011,   [X]补= 01011
        X=－0.1011,  [X]补= 10101

②整数补码的定义
















 

[X]补 =

 X    0≤X ＜2n
 2n+1＋X   － 2n ≤ X ＜ 0

例如：  X=+1011,   [X]补= 01011
        X=－1011,  [X]补= 10101

补码都是一样的，都是正数本身。补码是符号位为1，其它位是原码取反，未位加1。

补码的加法：001(+1)+111(-1)=000(0)

补码设计的目的：

**

⑴使符号位能与有效值部分一起参加运算,从而简化运算规则.

⑵使减法运算转换为加法运算,进一步简化计算机中运算器的线路设计

**

 

移码的定义：移码只用于表示浮点数的阶码，所以只用于整数

移码就是将符号位取反的补码。

Javascript中的类型详解

http://docs.jquery.com/Types是jquery的文档，主要讲了javascript的type。即使是js的老手看了也会有所帮助，现简要翻译过来！只翻译了通用的部分，jQuery专用的则不考虑。

STRING

 "I'm a String in JavaScript!"
 'So am I!'

Javascript中string是一个包含0或多个字符的不可变对象。对string类型进行typeof操作符操作得到的结果是”string”

Quoting

Javascript中可以用单引号或者是双引号来定义一个字符串。当用双引号来定义一个字符串时其内部可以中使用单引号来表示真正的“单引号”，反之则亦然。
如果使用双引号来定义字符串，那么要想表示真正的双引号则需要使用\来实现转义。

 "You make 'me' sad."
 'Holy "cranking" moses!'
 "<a href=\"home\">Home</a>"

Built-in Methods

string有很多的内建方法来操作字符串，返回的结果是一个新的string或者是其他（如split方法返回一个array）

 "hello".charAt(0) // "h"
 "hello".toUpperCase() // "HELLO"
 "Hello".toLowerCase() // "hello"
 "hello".replace(/e|o/g, "x") // "hxllx"
 "1,2,3".split(",") // ["1", "2", "3"]

Length属性

string的length属性表示字符串的长度。

 "Hello".length // 5
 "".length // 0

Boolean Default

一个空的string，在进行true or fasle判断时，代表false！

NUMBER

在javascript是双精度64位IEEE 754格式的值。和string类型一样，number也是不可变的对象。所有的基于C的一般操作符都能在javascript中使用（+, -, , /, %, =, +=, -=, =, /=, ++, —）。
number的typeof操作值是：

 typeof 12 // "number"
 typeof 3.543 // "number"

Boolean Default

如果number是0，那么就是false。

 !0 // true
 !1 // false
 !-1 // false

由于Number的实现方式是双精度64位IEEE 754格式的值，所以如下的情况不是bug。

0.1 + 0.2 // 0.30000000000000004

Math对象

Javascript中的Math对象提供特殊的数学操作：

 Math.PI // 3.141592653589793
 Math.cos(Math.PI) // -1

Parsing Numbers

parseInt和parseFloat方法被用来把string解析成numbers。如果没有明确指定的话它会进行一些隐式的转换：

 parseInt("123") = 123 (隐式转换为十进制)
 parseInt("010") = 8 (隐式转换为8进制)
 parseInt("0xCAFE") = 51966 (隐式转换为16进制)
 parseInt("010", 10) = 10 (显示转换为10进制)
 parseInt("11", 2) = 3 (显示转换为2进制)
 parseFloat("10.10") = 10.1

Numbers to Strings

当将number附加到string上时结果永远是string。数字相加和附加string的操作符都是“+”，所以要注意。

 "" + 1 + 2; // "12"
 "" + (1 + 2); // "3"
 "" + 0.0000001; // "1e-7"
 parseInt(0.0000001); // 1 (!)

当然你也可以用String方法实现转换：

 String(1) + String(2); //"12"
 String(1 + 2); //"3"

NaN and Infinity

当解析string成number时有可能会导致NaN。这是isNaN方法会帮助你探测这些情况：

 parseInt("hello", 10) // NaN
 isNaN(parseInt("hello", 10)) // true

以0来除一个除一个数会得到无穷大（Infinity）：

1/0; // Infinity

NaN和Infinity都是”number”类型的：

 typeof NaN // "number"
 typeof Infinity // "number"

注意NaN的比较是很奇怪的：

 NaN == NaN // false (!)

但是：

Infinity == Infinity // true

Integer

一个integer是一个普通的number类型。但是每当显示的提到了，表明一个非浮点型的number被期望了。

Float

一个float是一个普通的number类型。但是每当显示的提到了，表明一个浮点型的number被期望了。

BOOLEAN

在javascript中boolean类型可以是true或者false。

 if ( true ) console.log("always!")
 if ( false ) console.log("never!")

OBJECT

在javascript中所有都是object，虽然有些只是类似对象的。创建对象最容易的方法是使用对象符“{}”：

 var x = {};
 var y = {
   name: "Pete",
   age: 15
 };

object的typeof值是“object”：

 typeof {} // "object"

DOT操作符

你可以通过“.”操作符实现对象属性的读写：

 y.name // "Pete"
 y.age // 15
 x.name = y.name + " Pan" // "Pete Pan"
 x.age = y.age + 1 // 16

数组操作符

你也可以使用数组操作符“[]”来实现对象属性的读写：

 var operations = {
   increase: "++",
   decrease: "--"
 }
 var operation = "increase";
 operations[operation] // "++";
 operations["multiply"] = "*"; // "*"

迭代

可以在for-in循环中实现对象的迭代：

var obj = {
  name: "Pete",
  age: 15
};
for(key in obj) {
 alert("key is "+[key]+", value is "+obj[key]);
}

注意的是如果你修改Object.prototype，那么for-in循环也会显示你所添加的属性。

boolean default

一个object，无论它有没有属性，它永远都不是false:

!{} // false

Prototype

所有的对象都有一个prototype属。每当获取一个属性的时候，它也会检查prototype。jQuery使用prototype来为jQuery实体添加方法。

var form = $("#myform");
 form.clearForm; // undefined
 form.fn.clearForm = function() {
   return this.find(":input").each(function() {
     this.value = "";
   }).end();
 };
 form.clearForm() // works for all instances of jQuery objects, because the new method was added to the prototype

Array数组

Javascript中Arrays是一个可变的列表，并提供一些内建方法。你可以通过“[]”来定义数组：

 var x = [];
 var y = [1, 2, 3];

array对象的typeof值为”array”：

 var x = [];
 var y = [1, 2, 3];

你可以通过数组操作符来读写数组元素：

 x[0] = 1;
 y[2] // 3

迭代

你可以通过length属性实现迭代数组：

 for (var i = 0; i < a.length; i++) {
   // Do something with a[i]
 }

当性能需要较强时，可以对length属性进行缓存：

 for (var i = 0, j = a.length; i < j; i++) {
   // Do something with a[i]
 }

另外一种方式，定义变量来存储临时变量。它在数组中包含0或空字符串将不起作用：

 for (var i = 0, item; item = a[i]; i++) {
   // Do something with item
 }

length属性可以被用来添加属性到一个数组。这个push方法一样：

 var x = [];
 x.push(1);
 x[x.length] = 2;
 x // 1, 2

其他内建的方法：

 var x = [0, 3, 1, 2];
 x.reverse() // [2, 1, 3, 0]
 x.join(" – ") // "2 - 1 - 3 - 0"
 x.pop() // [2, 1, 3]
 x.unshift(-1) // [-1, 2, 1, 3]
 x.shift() // [2, 1, 3]
 x.sort() // [1, 2, 3]
 x.splice(1, 2) // [2, 3]

注意：.unshift()方法在IE下不反回数组的长度。

Boolean值

一个array不论它是不是length为0，它永远都不是false：

 ![] // false

Function

一个function可以是有名字的也可以是匿名的。如何函数都可以被赋值到一个变量中，也可以将函数变量传递到函数中。

function named() {}
var handler = function() {}

Arguments变量

在函数中有一个特殊的内部变量“arguments”永远可用，它和数组类似有一个length属性，但是它缺少数组的内建方法。这个伪数组变量代表了函数的参数变量：

function log(x) {
  console.log(typeof x, arguments.length);
}
log(); // "undefined", 0
log(1); // "number", 1
log("1", "2", "3"); // "string", 3

“arguments”对象有一个callee属性，这代表了它所在的function。例如：

 var awesome = function() { return arguments.callee }
 awesome() == awesome // true

Context,Call and Apply

在Javascript中，变量”this”代表了现在的context。默认，”this”对象代表了window对象。在function中this代表的context可以改变，依赖于这个函数如何被调用。
你可以通过call和apply函数实现function的调用和context的修改。

function scope() {
  console.log(this, arguments.length);
}
scope() // window, 0
scope.call("foobar", [1,2]); // "foobar", 1
scope.apply("foobar", [1,2]); // "foobar", 2

Scope作用域

在Javascript中，在function中定义的所有变量都只能在函数作用域中使用。考虑如下例子：

 // global
 var x = 0;
 (function() {
   // private
   var x = 1;
   console.log(x); // 1
 })();
 console.log(x); // 0

它在全局作用域中定义了一个x变量，又在匿名函数中定义了一个局部变量x。内部的x变量值在内部可见。

Closures闭包

每当一个外部作用域的变量在内部作用域被调用了，那么闭包就会被创建了。在如下的例子中，变量counter在create、increment和print函数中是可见的，但是它们不在外部。

function create() {
  var counter = 0;
  return {
    increment: function() {
      counter++;
    },
    print: function() {
      console.log(counter);
    }
  }
}
var c = create();
c.increment();
c.print(); // 1

这种模式能允许你创建一个方法来操作对外不可见的数据。这就是面向对象编程的基本。

代理模式

组合以上的知识可以给你作为一个javascript的编程人员很多可用途径。一种合并方法就是在javascript中实现一个代理模式，实现基本面向对象编程：

(function() {
  // log all calls to setArray
  var proxied = jQuery.fn.setArray;
  jQuery.fn.setArray = function() {
    console.log(this, arguments);
    return proxied.apply(this, arguments);
  };
})();

如上的包装代码的function实现了隐藏proxied变量的功能。它在闭包中存储了setArray方法并重写了它。代理就记录了调用了这个方法的context。使用apply(this,arguments)保证了调用被代理的函数。

Javascript中的“如果...那么...否则...”实现方法探究

在JS中，要实现“如果…那么…否则…”这一逻辑判断有两种方法，“if…else”或者是“? :”操作符。

具体实现：

“if…else”实现方法：

if (Math.round(Math.random()))
   doSomething();
else
   doSomethingElse();

“? :”操作符实现方法：

Math.round(Math.random()) ? doSomething() : doSomethingElse();

区别：

“if…else”实现相比“? :”操作符实现的代码可读性更高一点。

“? :”操作符实现所需要的字符少！这很可能是很多框架使用这种写法的原因之一，节省的字符也会积少成多（对于js来说大小也是很重要的）。

“if…else”实现能够包括更多的代码行，而“? :”操作符只能是一行（不过你可以将这些代码提取成函数，但是这样字符数就不会少了）。

性能的区别：

一直认为这两种写法性能的区别不大，但是今天测试了下，发现还是有些区别的。

测试代码（用到了jquery）：

    function doNothing(message) {
        $(“body”).append(“<div>”+message+”</div>”)
    }
    function testIfElse() {
        if(Math.round(Math.random()))doNothing(“testIfElse true”);else doNothing(“testIfElse false”);
    }
    function testAnd() {
        Math.round(Math.random()) ? doNothing(“testAnd true”) : doNothing(“testAnd false”);
    }
    function test(func,message,times) {
        var start = (new Date).getTime();
        times = times || 100;
        for (var i=0;i<times;i++) {
            func();
        }
        var diff = (new Date).getTime() - start;
        $(“<div>”+message+diff+”</div>”).appendTo(“#result”);
    }
    $(function() {
        test(testIfElse,”If Else take times:”);
        test(testAnd,”And take times:”);
    });

结果：

“? :”操作符实现的速度和大小都有一定的优势，一般情况下使用这个。但是如果是使用Javascript做一个大的应用，那么不建议使用这种方式。大的应用必然有很多人参与，如果代码晦涩难懂的话比执行速度慢更糟糕！

当然如果你需要包括很多代码行时，而这些代码行不能复用，不适合提取成一个函数，那么if…else仍旧是不错的选择！

另外：

“如果…那么…”这样的逻辑，除了if…else之外，也可以使用&&操作符实现。这个没有做过性能测试，不过从广大的开源库中频繁使用&&的情况来讲，应该与上面的情况类似。

JavaScript Math 对象详解

Math 对象并不像 Date 和 String 那样是对象的类，因此没有构造函数 Math()，像 Math.sin() 这样的函数只是函数，不是某个对象的方法。您无需创建它，通过把 Math 作为对象使用就可以调用其所有属性和方法。




































E返回算术常量 e，即自然对数的底数（约等于2.718）。
LN2返回 2 的自然对数（约等于0.693）。
LN10返回 10 的自然对数（约等于2.302）。
LOG2E返回以 2 为底的 e 的对数（约等于 1.414）。
LOG10E返回以 10 为底的 e 的对数（约等于0.434）。
PI返回圆周率（约等于3.14159）。
SQRT1_2返回返回 2 的平方根的倒数（约等于 0.707）。
SQRT2返回 2 的平方根（约等于 1.414）。












































































abs(x)返回数的绝对值。
acos(x)返回数的反余弦值。
asin(x)返回数的反正弦值。
atan(x)以介于 -PI/2 与 PI/2 弧度之间的数值来返回 x 的反正切值。
atan2(y,x)返回从 x 轴到点 (x,y) 的角度（介于 -PI/2 与 PI/2 弧度之间）。
ceil(x)对数进行上舍入。
cos(x)返回数的余弦。
exp(x)返回 e 的指数。
floor(x)对数进行下舍入。
log(x)返回数的自然对数（底为e）。
max(x,y)返回 x 和 y 中的最高值。
min(x,y)返回 x 和 y 中的最低值。
pow(x,y)返回 x 的 y 次幂。
random()返回 0 ~ 1 之间的随机数。
round(x)把数四舍五入为最接近的整数。
sin(x)返回数的正弦。
sqrt(x)返回数的平方根。
tan(x)返回角的正切。

如何实现随机获取true or false：

function random() {
    return Math.round(Math.random());
}

spring循环依赖导致死锁

一直让spring以为能导致循环死锁的原因是两个类都是用构造注入方法的可能性，如下：

@Service
public class A {
    private B b;
    @Autowired
    public A(B b) {
        this.b = b;
    }
}

@Service
public class B {
    private A a;
    @Autowired
    public B(A a) {
        this.a = a;
    }
}

以上情况是死锁的特例，一般也没人会这么写…

今天遇到了个类似问题不过更加复杂点。先看一个稍微复杂的的例子：

@Service
public class A {
    public A() {
        System.out.println("A inited!");
    }
}

@Service
public class B {
    @Autowired
    private A a;
    public B() {
        System.out.println("B inited!");
        System.out.println(a);
    }
}

最终打印结果是:

A inited!

B inited!

null

即使A先被初始化，B中打印出来的值还是null。可见装载生效是在B初始化完成之后的。（不过这很显而易见，因为没有初始化，又怎么去set呢！）这个例子不是关键！下面个例子十分特别，涉及到@PostConstruct注解。

由上例子可见无法在初始化方法中调用其他的bean，但是如果需要为bean做些初始化工作，那怎么办呢？这就需要用到@PostConstruct注解。它会在bean被装配完之后执行！很完美的解决了这个问题！

代码如下：

@Service
public class A {
    public A() {
        System.out.println("A inited!");
    }
    @Override
    public String toString() {
        return "a";
    }
}

@Service
public class B {
    @Autowired
    private A a;
    public B() {
        System.out.println("B inited!");
    }
    @PostConstruct
    public void init() {
        System.out.println(a);
    }
}

执行结果是：

A inited!

B inited!

a

 

执行正确！但是当情况在复杂一点，如下的代码：

@Service
public class A {
    @Autowired
    private B b;
    public A() {
        System.out.println("A inited!");
    }
    @PostConstruct
    public void init() {
        System.out.println("A post init!");
    try {
        Thread.sleep(100);
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
        System.out.println("A post inited!");
    }
    @Override
    public String toString() {
        return "a";
    }
}

@Service
public class B {
    @Autowired
    private A a;
    public B() {
        System.out.println("B inited!");
    }
    @PostConstruct
    public void init() {
        System.out.println("B post init!");
        System.out.println(a);
    }
}

执行的时候，奇怪的事情发生了，执行结果是：

A inited!

B inited!

A post init!

A post inited!

B post init!

然后进程死锁，停止运行！

结果令人匪夷所思，特别的是如果A的init方法sleep时间短一点，如10，那么不会发生这种事情！另外如果在sleep之前打印b，那么b正常输出而不是null(因为b已经初始化完成了)。

暂时想到的原因是A的init方法耗时较长，如果B在准备进入init时，A正在执行，则会导致B无法正常获取A的实例，导致死锁！

执行环境是spring 3.0 RELEASE版

Hibernate关键字is null在db2中的bug

有两个实体MessageLog和SendTask是OneToOne关系。MessageLog通过sendTask属性对应到SendTask的id，实现一对一关系。表的列名分别是：LOG.SEND_TASK_ID和SEND_TASK.ID.

示例KJAVA代码：

public Pagination queryPage(SendStatus status,int pageNumber, int pageSize) {
    StringBuilder hql = new StringBuilder("from MessageLog  where 1=1");
    List params = new ArrayList();
    hql.append("and (sendTask.sendFlag = ? or sendTask.id is null)");
    params.add(status);
    Long count = super.querySize(COUNT_ID + hql.toString(), params.toArray());
    return super.pageQuery(count, hql.toString(), params.toArray(), pageNumber, pageSize);
}

 

上面的代码看似没问题，实际上在oracle下执行没什么问题，但是放到db2下就有问题： 虽然执行条件是： 但是实际生效的部分只有

sendTask.sendFlag = ?

 

仔细看到hql被解析之后得到类似如下的sql：

Hibernate: select * from ( select rownumber() over() as rownumber_, messagelog0_.id as id43_, messagelog0_.content as content43_, messagelog0_.event as event43_, messagelog0_.receiver as receiver43_, messagelog0_.SEND_TASK_ID as SEND6_43_, time as time43_ from MHMOT.SMS_LOG messagelog0_ cross join XTT.T_SENDTASK sendtask1_ where messagelog0_.SEND_TASK_ID=sendtask1_.TASKID and 1=1 and (sendtask1_.sendFlag=? or messagelog0_.SEND_TASK_ID is null) ) as temp_ where rownumber_ <= ?

如果条件只是sendTask.id is null，得到的sql是：

Hibernate: select * from ( select rownumber() over() as rownumber_, messagelog0_.id as id43_, messagelog0_.content as content43_, messagelog0_.event as event43_, messagelog0_.receiver as receiver43_, messagelog0_.SEND_TASK_ID as SEND6_43_, time as time43_ from MHMOT.SMS_LOG messagelog0_ cross join XTT.T_SENDTASK sendtask1_ where messagelog0_.SEND_TASK_ID=sendtask1_.TASKID and 1=1 and (sendtask1_.sendFlag=? or messagelog0_.SEND_TASK_ID is null) ) as temp_ where rownumber_ <= ?

可以看到第一句sql多出了where messagelog0.SEND_TASK_ID=sendtask1.TASKID一部分，也就是说db2在此时去除了sendTask.id为null的情况，确保了sendTask.id必需存在。所以is null在db2下失效是因为在联接(join)两个表时，db2默认判断了联接用的列表项不能为null，所以导致了is null的失效。实际上，如果不使用表的联接，那么is null是可以正常工作的。

git使用记录

git是分布式的代码管理工具，远程的代码管理是基于ssh的，所以要使用远程的git则需要ssh的配置。

github的ssh配置如下：

一 、 生成SSH密钥过程：
1.查看是否已经有了ssh密钥：cd ~/.ssh
如果没有密钥则不会有此文件夹，有则备份删除
2.生存密钥：

$ ssh-keygen -t rsa -C “tekkub@gmail.com”
Generating public/private rsa key pair.
Enter file in which to save the key (/home/tekkub/.ssh/id_rsa):文件名
Enter passphrase (empty for no passphrase):密码
Enter same passphrase again:密码
Your identification has been saved in /home/tekkub/.ssh/id_rsa.
Your public key has been saved in /home/tekkub/.ssh/id_rsa.pub.
The key fingerprint is:
01:0f:f4:3b:ca:85:d6:17:a1:7d:f0:68:9d:f0:a2:db tekkub@gmail.com

最后得到了两个文件：文件名和文件名.pub
3.添加密钥到ssh：ssh-add 文件名
需要之前输入密码。
4.在github上添加ssh密钥，这要添加的是“文件名.pub”里面的公钥。
5.测试：ssh git@github.com

The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)? yes
Warning: Permanently added ‘github.com,207.97.227.239’ (RSA) to the list of known hosts.
ERROR: Hi tekkub! You’ve successfully authenticated, but GitHub does not provide shell access
Connection to github.com closed.

 

二、 设置git的user name和email：

$ git config --global user.name "Tekkub"
$ git config --global user.email "tekkub@gmail.com"

设置github的参数：

$ git config --global github.user tekkub
$ git config --global github.token 0123456789abcdef0123456789abcdef

github.user和github.token都可以从github上得到。

 

三、 开始使用github
1.获取源码：

$ git clone git@github.com:billyanyteen/github-services.git

2.这样你的机器上就有一个repo了。
3.git于svn所不同的是git是分布式的，没有服务器概念。所有的人的机器上都有一个repo，每次提交都是给自己机器的repo
仓库初始化：

git init

生成快照并存入项目索引：

git add

文件,还有git rm,git mv等等…
项目索引提交：

git commit

4.协作编程：
将本地repo于远程的origin的repo合并，
推送本地更新到远程：

git push origin master

更新远程更新到本地：

git pull origin master

补充：
添加远端repo：

$ git remote add upstream git://github.com/pjhyett/github-services.git

重命名远端repo：

$ git://github.com/pjhyett/github-services.git为“upstream”
