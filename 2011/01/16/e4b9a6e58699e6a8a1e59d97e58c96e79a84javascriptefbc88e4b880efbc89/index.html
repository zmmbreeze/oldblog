书写模块化的Javascript（一）

  1. Javascript是一种非常灵活的语言；
  2. Javascript可以实现函数式编程的风格；
  3. Javascript的原型机制太棒了；
  4. …
    你可以举例出Javascript很多的优点，但你发现Javascript有一个致命的缺点，没有原生支持的模块化组织方式。原因是JS至今为止都是一门运行在浏览器上的语言，很少会用它来做主要开发语言。放眼后端的编程语言，没有一个好的包管理方式，都不好意思出来混。不过情况正在改变，已经出现很多的标准实现JS的模块化：Standards and proposals for JavaScript Modules and jQuery、AMD vs Wrappings。本文要说的不是这些高深的技巧，而是在这些标准下如何写一个可靠的模块，模块内部也需要好的组织方式。

作为一个菜鸟写的代码：

[js]
moduelName = {
func_1:function() {

},
func_2:function() {

},
pub_var:1,
pub_var:2

}
[/js]

这样写模块的缺点是module没办法定义私有变量。而且模块在加载之后就立即执行了，这无法实现JS执行和下载分开。这一切都可以用function wrap代码，使其在下载之后不会立即执行！所以得到如下修改：

[js]
var moduleName = (function() {
// private var
var private_var;
// private function
function private_func_1() {
tmp_var = 1;
}
function private_func_2() {
tmp_var = 2;
}
// init code
// public var and function
return {
pub_var:1,
pub_init:function() {
pub_var = 2;
private_var = 1;
}
};
})();
[/js]

以上代码看起来已经不错了，但还有一个问题：

初始代码代码一旦执行，就无法再使用第二遍。例如：tab这样的组件可能会在同一个页面上使用两次，所以写成匿名函数直接执行，使得初始化代码只能执行一次，私有变量只有一份是欠妥的。所以还需要再修改下：

[js]
var moduleName = (function() {
// private var
var private_var;
// private function
function private_func_1() {
tmp_var = 1;
}
function private_func_2() {
tmp_var = 2;
}
// init code
// public var and function
return function() {
// private var for every acture module;
var private_var_for_every_module;
// init code which can be use again;
this.pub_func = function(){};
this.pub_var = 1;
};
})();
[/js]

这下一我们的模块不是直接的单个对象而是类，可以通过new的方式获得多个对象。

不过我们还没考虑到模块的继承与复用！考虑到一半的代码混淆工具会自动压缩变量，所以把外部变量当作形参传入还可以减少字节数。修改后得到：

[js]
var subModuleName = (function(parent) {
var parent = $.isFunction(parent) ? new parent : parent, // 如果父模块是类则是用new
private_var; // private var
// private function
function private_func_1() {
tmp_var = 1;
}
function private_func_2() {
tmp_var = 2;
}
// init code
// public var and function
parent.pub_var = 1;
parent.pub_func = function() {
};
return parentModel; //如果不是继承只是复用，那么依旧用新建一个类或对象的方式来暴露公有API
})(parentModel || {});
[/js]

最后加上jQuery完善下得到最终的两个结果：

[js]
var moduleName = (function(argument/常量/, $, parentModuel, otherModule2) {
var parent = $.isFunction(parentModel) ? new parentModule : parentModule;
// private var
// private function
// init code
// public var and function
//第一种选择，直接返回对象
return {};
//第二种选择，返回初始化方法，可生成多个对象
return function() {
// private var for every acture module;
// init code which can be use again;
// use this set public var
// use this set public function
};
//第三种选择，继承父模块
return parent;
})("arg", jQuery, parentModule || {}, otherModule2 || {});
[/js]

模块的载入方法不在这里讨论,下一篇文章继续。
