模块化CSS

原来写过一篇：更好的组织CSS，里面有一些我当初对于CSS写法的想法。现在这篇文章有些旧了。当初的想法虽然依旧可用，但是有了些改变。于是决定再写一篇。

CSS是完全没有逻辑的语言，可以说它不适合于团队开发。但是我们可以通过一个好的写法避免团队之间的命名冲突、相互依赖等等问题。

注：为了尽量避免js和css之间的开发关联，js的钩子基本采用id，且使用驼峰命名，不得已才会用class。CSS全部使用class作为选择符（不用id），且不适用驼峰命名方式，下面会详细解释class的命名方式。

CSS文件的分布

在弹css的写法之前，先说下整个网站的CSS分布。每个页面的CSS文件不多于两个，其中一个为全局CSS，一个为局部CSS。全局CSS为整个网站所有页面都会引用，但是局部CSS只会被相似的页面引用（或者是同一个模块的页面）。这么做是平衡了缓存和开发上的考虑。太集中不利于开发更新，太分散不利于文件缓存。

CSS模块化

所谓模块就是在UI显示上为一体的多个CSS样式。简而言之就是：它们在网页上是仅靠的一坨东西（玩笑～）。对于这样的多个CSS样式，我们取一个“根class名”，然后对于其下级元素的class名，使用“-”符号分割。例如：

[html]
<div class="blackboard">
<div class="blackboard-title">
<h2>黑板报</h2>
</div>
<div class="blackboard-content">在宇宙的元初，当一片星尘与另一片星尘相遇，凝聚成核；在生命的元初，当一团原生质和另一团原生质结合，产生新的生命；在文明的元初，当一个尼安德特人和一个智人在荒原上四目相对，又怅然离去；在万物寂寞的元初，万老师说要有引力——在这里你们要相信，你们来到世间的使命就是彼此吸引。现在，让定律奔跑，让宇宙运转，让我们开始遇见，让超新星嫉妒到爆炸。</div>
</div>
[/html]

它的“根class名”为“blackboard”，子元素的class名则都以“blackboard-”开头。于是乎在写css的时候就可以写：

[css]/ @module: blackboard 黑板报/
.blackboard {
margin-bottom: 20px;
overflow: hidden;
width: 100%;
}
.blackboard-title {
margin-bottom: 10px;
width: 100%;
overflow: hidden;
}
.blackboard-content {
padding: 20px;
color: #333;
word-wrap: break-word;
background-color: #F7F6EC;
}[/css]

如上所示，利用四个空格的缩减在代码上区别层级，这样就可以利用编辑器的自动折叠功能，例如Vim。

每个模块都有自己的“根class名”，只要确保“根”的名字不同，那么就会很好的避免命名冲突。利用写好的注释，只要在即将写入的文件中搜索“@module”，就可以看到所有模块的“根”了。

模块的分类

CSS模块分为三种：全局模块、局部和组件。全局模块是所有的页面都会用到的模块，例如网站导航nav等等，它存放于全局css文件中。局部模块是只有部分页面会用到的模块，存放与局部css文件中。

CSS组件

组件的存放位置不定，它是其他模块中的一部分。通常是一种通用的样式。例如通用按钮、通用标题。它是组成其他模块的一部分。如下是一个左图右文的通用组件：

[css]
/ @module: pt 左图右文模块，图为48px /
.pt {
}
.pt-pic {
_display: inline;
float: left;
margin-right: 12px;
}
/主标题/
.pt-txt h3 {
display: inline;
}
/副标题/
.pt-txt span {
color: #999;
}
/内容/
.pt-txt-d {
margin-top: 4px;
}[/css]

其他的模块也会用到这个组件，例如：用户列表模块用到了“左图右文”组件：

[html]
<ul class="groups">
<li>
<a class="pt-pic" href="#" target="_blank"><img src="http://www.guokr.com/gkimage/90/b3/q6/90b3q6.png&quot; alt="" /></a>
<div class="pt-txt">
<h3><a href="#" target="_blank">新房</a></h3>
<span>203700000人关注</span>
广东医学院人体解剖学教师<a class="sai" title="果壳达人" href="/nuts/">ψ</a>
</div>
</li>
</ul>
[/html]

上面的小组列表模块的“根class”是“groups”，它用到了“pt”组件，所以里面的class命名不是以“groups-”开头的。

其他命名方式相关问题

除了之前说的模块中有组件的情况，还一种不以“根class-”开头来命名子元素class的情况是表示状态的时候。例如tab列表：

[html]
<ul class="nav">
<li><a class="current" href="/">首页</a></li>
<li><a href="/minisite/">主题站</a></li>
<li><a href="/group/">小组</a></li>
<li><a href="/nuts/">果壳达人</a></li>
<li><a href="/ask/">问答</a></li>
</ul>
[/html]

当前选中tab的高亮状态用“current”做为class名字区分！之所以这么做是为了兼容以前的写法，当然不得不承认“nav-current”的写法有些冗长。

这样我们就不得不列一个公用状态class名，以避免重复！例如：“current、now、hover、active”。

最后要说的是，为了避免出现“header-right-navgation-button”的class名字，简写和好的语义化标签是需要的。这样就简写成“header-r-nav-btn”。

如果语义化做的好还可以做到：“.header-r .header-r-nav {} ” 变成 “.header-r ul {} ”

注：根元素不要使用简写，是为了减少命名冲突。

开发时的CSS文件分离

虽然每个页面的CSS文件只能有两个，但是在开发时我们可以通过CSS的@import，将一个文件分成多个。但是不能分太细，因为IE6对@import文件限制。所以推荐以下的分离方法。

全局文件按照全局模块分离成为多个文件，并且将所有全局组件独立成为一个common.css文件。文件及目录结构如下：

g.css

g/common.css

g/header.css

g/footer.css

g/reset.css
g.css的文件内容就是5行import既可！

局部文件则按照页面分离成为多个文件。文件及目录结构如下：

mini.css

mini/ask.css / 问答页面 /

mini/minisite.css / 主题站页面 /

mini/post.css / 帖子页面 /
mini.css文件除了要import另外三个文件之外，还可以写这三个页面的通用样式模块。

当然线上环境是不应该使用@import的，所以需要一个合并压缩工具。在提交测试之前将文件合并然后压缩，这样对于线上环境文件数量并没有改变。
