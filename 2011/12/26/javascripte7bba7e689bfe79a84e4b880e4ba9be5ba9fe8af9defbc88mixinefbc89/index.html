Javascript继承的一些废话（Mixin）

之前在Prototype库面向对象的实现原理一文里面写过一些关于Javascript实现OO的方法。一些主流的库的OO也都是这么实现的：

  1. Prototype
  2. Klass
  3. MooTool
    不过这种模式有些缺点：

修改父类prototype的属性影响到子类的属性

代码（基于MooTool）：

[js]
var C = new Class({});
var B = new Class({Extends: C});
C.prototype.name = ‘1’;
new C().name;
new B().name;
[/js]

当C的prototype值修改的时候，B的实例也会受到影响。

类的constructor可以通过for in遍历到

因为ECMAScript 3的设计，以及这种模式需要显示设置constructor值。所以for in循环是可以遍历到constructor值的！

查找属性时原型链层数可能太深

[js]
var Super1 = new Class({name: ‘Super Name’});
var Super2 = new Class({Extends: Super1});
var Super3 = new Class({Extends: Super2});
var Super4 = new Class({Extends: Super3});
var Son = new Class({Extends:Super4});
new Son().name
[/js]

最后一行在访问name的时候，需要查找原型链的层数有5层，如下chrome终端中的结果：

如果name属性是频繁调用的，那么对性能的影响很大。

Mixin

目前最好的方法使用Mixin方式解决继承，它可以规避上面存在的三个问题。而且它的代码很简单，更像javascript而不是其他更擅长OO的语言。

[js]
function Super() {}
Super.prototype.a = function () {console.log(‘a’);};
Super.prototype.b = function () {console.log(‘b’);};
function Son() {}
for (var method in Super.prototype) {
Son.prototype[method] = Super.prototype[method];
}
Super.prototype.c = function () {console.log(‘c’);};
var son = new Son;
son.a(); // ‘a’
[/js]

Mixin使用拷贝属性值的方式替代了原型的继承方式，更好的保证了：

  1. 父类的prototype修改后不影响子类
  2. 不需要设置constructor
  3. 不增加原型链层数
    所以Mixin方式更适合Javascript，而且在构建复杂的web应用的时候更快！

Proto.js

这是一个基于mixin实现OOP的Javascript库：地址。下面是它的使用样例：

[js]
// Person
// Method: Proto.$extend
// @param {function} construtor [option]
// @return {function} Class
var Person = Proto.$extend(function(supr, name, age) {
this.age = age;
this.name = name;
});
// Method: $methods
// @param {object} map methods to be added
// @return {function} Class
Person.$methods({
say: function(supr) {
console.log(‘I\’m a Person.’);
console.log(‘My name is ‘+this.name+’.’);
console.log(‘I\’m ‘+this.age+’ years old.’);
}
});
// Method: $statics
// @param {object} map static methods to be added
// @return {function} Class
Person.$statics({
doSomething: function(supr) {
console.log(‘Do something!’);
}
});

// Student extend from Person
var Student = Person.$extend();
Student.$methods(‘say’, function(supr) {
// use supr to call Person’s method
// must pass this to supr
supr(this);
console.log(‘And I\m a Student.’);
});
Student.$statics(‘doSomething’, function(supr) {
// use supr to call Person’s static method
// supr(this);
return ‘OK’;
});
[/js]
