Fuck Javascript之二：奇怪的true/false

ture or fasle是判断时必须要用的，javascript作为弱类型语言。知道什么样的变量值能得到false是很重要的。

判断时是false的值：0, “”, null, undefined, NaN.

上面的只是基础知识，下面看javascript能给我们带来什么惊喜！

if (-2 < “-1”) {
    console.log(“sss”);
}

结果是打印出了“sss”，可见比较之前先转换成了number类型！

if (“ssss” > “ssss”) {
    console.log(“bbb”);
}
if (“-2” > “-1”) {
    console.log(“aaa”);
}
if (-2 > -1) {
    console.log(“ccc”);
}

结果是“aaa”，问题来了！如果说转换成了整数，那么为什么”‘-2’>’-1’ === true”!再看：

if (“-2” < “-1”) {
    console.log(“aaa”);
}
if (“4” > “3”) {
    console.log(“bbb”);
}
if (“-2” < “1”) {
    console.log(“ccc”);
}

能够打印出“bbb ccc”，负号丢失了！如果有一个为正值或0，另一个为负值那么还是正值大！

上面只是个开始，还不止这些。
看如下情况：

if (“c” > “b”) {
    console.log(“aaa”);
}
if (“a” < “b”) {
    console.log(“bbb”);
}

可见是字符串是是按照字母顺序排序，a最小，z最大！

那么数字和文字夹杂的情况呢？

if (“12a” < “12b”)
    console.log(“12a<12b”);

if (“13b” > “12b”)
console.log(“13a>12b”);
得到的结果是是12a<12b, 13a>12b.可以判定按从左至右的顺序对每一对char判断大小，直到能决定出结果。

那么特殊字符呢？显然其判断并不像上面所讲述的那么简单。看如下：

if (“@” < “a”)
    console.log(“@<a”);

打印结果为”@<a”。后来发现，其实是根据字符的编码进行判断。javascript使用unicode编码（扩展了ASCII），根据ASCII编码@为64、a为97。同时这也解释了为什么“-2”<”0”,因为ASCII中-为46，而0为48。一切都有了合理的说法：

string类型的比较是按从左至右的顺序对每一对char，根据其编码的对应值判断大小。直到能决定出结果！当number和string比较时string会被转换成number，然后比较。如果转换不成则返回false。
string看完了，那object呢？看如下情况：

var o1 = {toString:function(){return “1”;}};
var o2 = {toString:function(){return “2”;}};
if (o1 < o2)
    console.log(“o1<o2”);
if (o1 > o2)
    console.log(“o1>o2”);

得到的结果是“o1<o2”。可见object会先被转换成string（使用toString方法），然后进行判断。

似乎一切都有了结果，那么看如下：

if (undefined < 2) {
    console.log(“sss”);
}

不能打印“sss”，”undefined < 2”这能够比较吗？应该报错吧？假设能够比较那么结果是undefined比2大，那么在比：

if (undefined > 2) {
    console.log(“sss”);
}

依旧不能打印！等你试完所有可能性：”2 < undefined”,”2 > undefined”，依旧不能打印“sss”。

这就是弱类型语言的特性，在”undefined < 2”比较时，发现undefined值并且两者无法比较之后，不再考虑其大小，直接返回了false。

可见对于undefined这样特殊值有特殊的处理，他们不会被转换成string或是number来比较，据测试null，NaN也是一样！

那么：

if (true < 2)
    console.log(“true < 2”);
if (true == 1)
    console.log(“true == 1”);
if (false == 0)
    console.log(“false == 0”);
if (false < true)
    console.log(“false < true”);
if (false < “e”)
    console.log(“false < \”e\””);
if (false < “2”)
    console.log(“false < \”2\””);

所有的结果都被打印了，可见true被转换成了1,false被转换成了0。

最后还有””,测试后发现它比所有的string都小！


总结：到底“<,>等”比较操作符做了哪些事情？

  1. 如果有任何一个值是undefined,NaN,null则返回false；
  2. 如果有任何一个值是true，false,则将他们转换成1和0；
  3. 如果有值是number类型(true/false转换之后的1/0也算)，那么转换另一个比较值成number；如果有值是string类型，那么转换另一个比较值成string；
  4. 转换成功则比较，转换不成则返回false；
  5. 将两个值转换成string值根据编码值由左至右比较，直到分出大小。如果有一个值是“”,那么它相比其他字符串最小；
    那么这种情况能给我带来什么编程上的方便呢？
    // 让 UA.ie 的默认值是 undefined, 在 ie 下，UA.ie = ie 的主版本号
    // 这样，
    if(UA.ie < 8) {
    // 针对 ie6 和 ie7 的代码
    }

    以上代码，判断了UA.ie存在且值小于8的情况，这节省了判断浏览器是否是IE的代码！

遍历数组:

//normal way
var array = [1,2,3,4];
for (var i=0;i<array.length;i++) {
   var value = array[i];
   //todo
}

//javascript way
i=0;
while( (value = array[i++]) ) {
//todo
}
对于javascript,作用域按照function来算,所以不存在变量i占用较大作用域的问题! 新方法节省了不少代码,并且减少了判断i<array.length的步骤!不过有一定局限性,必须确保array的每个值不能是0,””,undefined,null,NaN,false!
