浏览器记住密码引发的用户体验问题

很久不写Blog了，因为最近做的事情大都不让自己很满意。积压了不少草稿都写到一半，因为发现自己的解决思路不可行。本篇最终也没有提出一个令我满意的解决方案来，不过最后还是有一个基本满足需求的方案。

现代浏览器一般都会带有表单的自动补全功能！这极大的方便了用户，让他们不用启用cookies记住登陆状态就能记住登录用户名与密码直接登录。但是出于安全情况的考虑为了避免密码在网络上明文传播，我们需要做的就是在客户端对密码进行加密，然后由服务器端解密之后使用。

看一个简单的示例：

html代码：

[html]
<form action="." method="POST">
<input type="password" name="pwd" id="pwd"/>
</form>
[/html]

JS代码：

[js]$("form").submit(function() {
var pwd = $("#pwd");
pwd.val(encrypt(pwd.val()));
})[/js]

给form绑定一个submit事件来实现在提交表单之前加密的功能。在IE和Firefox 3.6这样的提供表单自动完成功能的浏览器中这一切看起来很完美。

但是在Chrome中悲剧出现了，Chrome记住的居然是加密后的密码，同样的问题发生在Safari中！（注：所有的浏览器都是在win7下测试的，没有发现Opera记住密码，不知是何原因，有待查明）。

很显然，问题在于记住密码这个动作浏览器是在何时执行的。IE和Firefox记住密码的时间点在于submit事件发生之后绑定的监听方法执行之前。而Chrome和Opera则是在绑定的监听方法执行之后post数据之前。

显然对于Chrome和Safari用户来说：登录时输入框中突然多出一堆莫名其妙的数字是很不好的体验。

所以考虑了如下的解决方案：

html代码：

[html]
<form action="." method="POST">
<input type="password" name="pwdFake" id="pwdFake"/>
<input type="hidden" name="pwd" id="pwd"/>
</form>
[/html]

JS代码：

[js]$("form").submit(function() {
$("#pwd").val(encrypt($("#pwdFake").val()));
})[/js]

提供一个隐藏的输入框来传输密码，这样浏览器便可以在Chrome和Safari下实现记住正确密码的功能了。

万事大吉了？不没有，回想下最初的目的不就是为了避免密码明文传输吗？悲剧的是你的密码换了身马甲在网络上大摇大摆的轧马路。

我曾经试过将pwdFake放在form外面，但是所有的浏览器都不会记住这个密码了，因为这个“记住密码”的功能真名叫“表单自动填充”，既然不在表单之中又何必要记住你呢？

所以经过一些尝试之后，为了避免pwdFake被当作参数传输出去，考虑了如下方案：

Html代码不变。

JS代码：

[js]$("form").submit(function() {
var fake = $("#pwdFake");
$("#pwd").val(encrypt(fake.val()));
fake.attr("disabled","disabled");
})[/js]

对于Safari和其他浏览器来说就可以了，密码不会传播，只会记住加密前的密码。可以Chrome仍旧不同意，它不会记住不发送参数。可见Chrome的记住密码可以说相当“深入”，可见Chrome的开发工程师为了节省那点硬盘空间费了很大的周折……

好吧！正规方法不可以，只能对Chrome进行hack了。详细分析一开始的需求：密码不能明文传播同时浏览器自动记录并不全未加密密码。既然Chrome支持Html5，不如使用离线存储实现记住密码的功能，请出“localStorage”登场了！

[js]$(function(){
var fake = $("#pwdFake");
if (isChrome && window.localStorage && localStorage.pwd) {
fake.val(localStorage.pwd);
}
$("form").submit(function() {
$("#pwd").val(encrypt(fake.val()));
fake.attr("disabled","disabled");
if (isChrome && window.localStorage) {
localStorage.pwd = fake.val();
}
});
});[/js]

终于Chrome实现了“记住密码”，不过不是浏览器原生的。 不过问题又来了，这样的存储是否安全呢？明文存于本地是否会造成其他的后果？或许我们可以考虑在本地加密、解密。目前还没想到最佳方案，待定…

2010/11/25日更新

发现了解决chrome把明文密码上传bug的解决方案。

[js]$(function(){
var fake = $("#pwdFake");
$("form").submit(function() {
$("#pwd").val(encrypt(fake.val()));
if (isChrome && window.localStorage) {
fake.val("");
}
else {
fake.attr("disabled","disabled");</pre>
}
});
});[/js]
Chrome浏览器下设置pwdFake为空就能让它记住未加密的密码，这方法同样适用于除safari之外的浏览器。对于safari浏览器可以使用disabled的方法来实现阻止明文密码的发送。至此问题解决！
