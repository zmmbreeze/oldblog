使用window.name来实现跨域

前端跨域请求算是这个领域老生常谈的必修课之一了。前辈们已经摸索出了很多有用的方法：

  1. JSONP
  2. 同根域名下的修改document.domain
  3. iframe + 修改location.hash
  4. iframe + proxy页面
  5. iframe + window.name方式
  6. HTML5 postMessage方式
  7. flash
  8. 发伪img请求，传输数据（很多日志类请求用此方法）
  9. …
    每个方法都有各自的优缺点。这里要谈的是“iframe + window.name”的方式。它利用了window.name属性的特性：“一旦被设置，即使在页面重新定位之后也不会改变”。如下是大概流程：

关键点在于数据页面会把数据转成字符串设置到window.name上，然后再把iframe定位到proxy页面。proxy页面和主页面是同一域名下，故主页面可以访问proxy页面的window.name。

这个方法有如下的优点：

  1. 支持POST和GET
  2. 所有主要浏览器都支持
  3. 支持传输的数据量大（2M）
  4. 无需任何插件

  5. 机制不算复杂（比起flash和location.hash之类来说要简单些）
    当然它也有些缺点：

  6. 需要同源的proxy页面

  7. 需要请求一次proxy页面
  8. 需要创建和销毁iframe，相比JSONP和img请求之类的方法，开销要大些
    我写了一个DEMO。一开始我以为这个方法没有什么兼容性问题，但是实际使用的时候还是发现了一个。

    IE10下，一旦设置iframe.name之后，再在iframe内部设置window.name时只能在iframe内部访问。在iframe外部调用iframe.contentWindow.name获取到的值是iframe.name而非window.name
    用GET提交数据时，这不会有问题。但如果需要使用POST，那就需要构造form来提交数据。这时一般会用到“form.target与iframe.name相同，来返回页面到指定的iframe”这一技术。于是一旦设置iframe.name之后，利用window.name来跨域的方法就会失效了。试了几种方法，都没找到合适的hack。最终只能把form构造在iframe内部，虽然麻烦些，但是可以做到主流浏览器都兼容。

网上还有window.name + postMessage互补的方式来解决这个问题，优点是在高级浏览器上可以减少一次请求（proxy文件），缺点是前后端代码都稍微变得复杂些。目前已经有了很多成熟的方案了，可以自行Google。

OAuth入门，让你的网站连接新浪微博

新浪微博拥有两种认证方式：OAuth、Http Basic Authoration。OAuth因为不需要将用户的密码存于第三方网站而更安全，被大多数web应用采用。不过Basic Authoration也因为更简单容易操作所以适合本地应用。

闲话少说，根据http://open.t.sina.com.cn/wiki/index.php/OAuth的描述。整个OAuth的流程如下：

要想让自己的网站与新浪微博链接起来，需要如下步骤（以土豆为例子）：

  1. 在登录页面放置“新浪微博按钮”，http://login.tudou.com/login.do?service=http://www.tudou.com/&noreg=ok；
  2. 在网站端提供跨域代理，获得包含oauth请求的token，如：http://login.tudou.com/xiaonei/sina/auth.action；
  3. 点击按钮，用iframe显示这个链接；
  4. 此链接会返回302响应，重定向到：http://api.t.sina.com.cn/oauth/authorize?oauth_token=33a3883d7c121c397b29755305f65a4a
  5. 这个链接会显示新浪的登录表单，用户登录后表示用户授权然后重定向到事先设置好的callback_url。如豆的链接是：http://login.tudou.com/xiaonei/sina/lo.action?oauth_token=475c264a22b3aae92b59046f1868c12f&oauth_verifier=173444；
  6. 这个链接会记录用户的登录状态，生成一个新的帐号，并且由js控制重新定向到土豆首页。土豆在这个页面的内容是：
    [js]
    <script>
    document.domain = ‘tudou.com’;
    (function() {
    if(window.syc) return;
    window.syc = true;
    parent.loginSina("http://login.tudou.com/login.do?act=newreg&user=_78484255&code=47g7j2mgafc71o0l4gj9&service=http://www.tudou.com&quot;);
    })();
    </script>
    [/js]

这样就完成了用新浪微博账户登录的流程。主要涉及到跨域的步骤是第一步和第六步。第一步使用服务器端做代理的模式避免了直接去跨域请求。第六步使用了iframe方式，通过设置domain实现了跨域。
