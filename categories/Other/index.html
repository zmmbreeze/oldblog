抛弃SyntaxHighlighter Evolved插件

代码高亮可能是wordpress里面最麻烦的一件事件了，以前一直用的插件叫做：SyntaxHighlighter Evolved。但是说实话，这个插件不太好用…编辑器在可视化和文本之间切换容易造成bug（代码无故加上<pre>之类的文字）。最悲剧地是：由于SyntaxHighlighter Evolved的语法很特殊，没办法替换成其他的高亮插件。语法如下（去掉空格）：

[[js]]code[/ js]
不过这不是我替换掉它的主要原因。致命的缺陷是它使用的高亮js库是：[SyntaxHighlighter](http://alexgorbatchev.com/SyntaxHighlighter/)。这个库已经有些年头了，它生成的html是table而不是pre标签。table很难用CSS来优化，且不符合语义化。 于是fork修改了[SyntaxHighlighter Evolved的代码](https://github.com/zmmbreeze/syntaxhighlighter)，去掉了SyntaxHighlighter相关的html输出。这样就可以保证旧的文章依旧输出正确的pre标签。然后在主题里面使用[google-code-prettify](http://code.google.com/p/google-code-prettify/)来高亮代码。google-code-prettify有个很重要的优势就是可以自动判断代码的类型，这样对任何pre标签做处理即可。代码如下： 
(function() {
    var pres = document.getElementsByTagName('pre');
    for (var i=0,l=pres.length; i<l; i++) {
        pres[i].className = 'prettyprint';
    }
    prettyPrint();
})();

几个编码习惯的改变

JS的编码习惯：

1. 使用单引号来写字符串

原来一直是遗传了java中的写法用双引号来写字符串，但是在Javascript中使用双引号有个缺点。

[js]

var html = "<div class=\"block\"></div>";

var html = "<div class=’block’></div>";

[/js]

如果用string来表示html，那么需要在html里面写单引号。但是在一般的html写法中习惯使用了双引号。必须要用这两种方法中的一种，但是第一种会多出多余的字符，第二种方法会导致js中的html和html文件中的写法不一样。所以最近改变了习惯，用单引号写字符串。

CSS的编码习惯

1. CSS的选择符使用”-“来分割

之前的文章“更好的组织CSS”提到过用模块化的CSS选择符。

[css]

.side_login{ width:260px;overflow:hidden;}

[/css]

之前有段时间一直使用了下划线’_’来分割不同的“域”，但是实际上下划线并不是用来分割单词的，如果一个单词内部还有划分就找不到合适的符号。所以我使用了中线来代替，下划线用来分割一个单词内部的两个不同语义。例如：

[css]

.side-login_box{ width:260px;overflow:hidden;}

[/css]

另外一个好处在于编辑器方面：如果使用中线来分割单词，当你双击单词时，自动选中的是其中的一部发。但是如果是下划线会自动选中全部。在搜索时，往往是要搜一个“域”（即单词中的一部分，因为那部分是最容易重复的）而不是全部。

2. 使用class做选择符而不是html tag

先看如下例子：

[html]
<div class="control">
<span>test1</span>
<a href="#">a1</a>
<span>test2</span>
<span>test3</span>
</div>
[/html]

这样的html要写CSS的话，得到如下结果：

[css]
.control span{color:red;font-size:12px;}
.control a{color:blue;font-size:12px;}
[/css]

对于这样很容易修改的网页部分需求很容易改变。如果要添加了一个新的部分或者是只修改最后一个span，那么需要修改CSS：

[css]
.control span{color:red;font-size:12px;}
.control a{color:blue;font-size:12px;}
.control-last{color:yellow;font-size:14px;}
[/css]

单单只有这样一个修改很容易，但是如果span有很多，修改的部分也很多，那么会导致很多的冗余，例如：

[css]
.control span{color:red;font-size:12px;border:1px solid #000;background:url(imgs/add.png);}
.control a{color:blue;font-size:12px;}
.control-last{color:yellow;font-size:14px;border:none;background:none;}
.control-first{color:yellow;font-size:13px;border:none;background:none;}
.control-second{color:yellow;font-size:15px;border:none;background:none;}
[/css]

可以发现每个特殊样式会有很多的冗余：border和background。并且这样也会很容易在修改添加的时候会出现一项不到的bug。
所以说，不要给html tag的选择符添加无法继承的样式，类似与border,background这样的样式需要设置重置。如果是可以继承的样式那么倒是没什么关系。

3. 使用tab来表示CSS中的作用域

在CSS中没有原生的作用域支持，并且更重要的是编辑器不会提供代码折叠。这在修改重构CSS的样式会很难找到对应的选择符。
所以不如自己用tab来表示作用域，例子如下：
[css]
.control{width:260px;overflow:hidden;}
    .control span{color:red;font-size:12px;border:1px solid #000;background:url(imgs/add.png);}
    .control a{color:blue;font-size:12px;}
    .control-last{color:yellow;font-size:14px;border:none;background:none;}
    .control-first{color:yellow;font-size:13px;border:none;background:none;}
    .control-second{color:yellow;font-size:15px;border:none;background:none;}
[/css]

 这样虽然没有折叠，但是也可以达到一定程度的达到帮助浏览代码的目的。

4. 以内容本质来起class名字，并使用组合方式表示相同样式，而不是给它们一个相同的class

当多个不同的元素拥有多个相同的样式的时候，以前我的做法是给一个相同的class(以样式来命名)，然后统一设置。后来发现这样的可维护性很差，因为虽然暂时这多个元素具有相同的样式，但是实际上这几个元素的内容是不同！以后在进一步的改版过程中，因为内容不同所以本质就不同，导致了这几个样式可能会有不同的变化，此时要么用新的class替换旧的，要么就添加一个新的class。两者的维护性都不好。所以不如一开始就以内容本质来命名class，然后用组合的方式写CSS。如下： [css]
.side-article,
.side-ad,
.side-name,
.side-ask{
margin-bottom:10px;
width:260px;
overflow:hidden;
}
[/css]

十进制与二进制转换

纯为复习一下写下此文！

整数十进制转二进制：迭代除2，直至商为0，所有的余数即是此二进制(有后向前排列)。

例如：101

101/2 = 50 余 1

50/2 = 25 余 0

25/2 = 12 余 1

12/2 = 6 余 0

6/2 = 3 余 0

3/2 = 1 余 1

1/2 = 0 余 1

二进制为：1100101
整数二进制转十进制
例子：1100101
12E6 + 12E7 + 12E2 + 12E0 = 101
小数十进制转二进制：迭代取小数部分乘2，直到得到的结果小数位为0。每次相乘的整数位为1时记此二进制为1；

例子1：

0.325 *2 = 0.65 (取0)

(0.65 – 0) * 2 = 1.3 (取1)

(1.3 - 1) x 2 = 0.6（取0)

(0.6 - 0) x 2 = 1.2 (取1)

(1.2 - 1) x 2 = 0.4（取0)

(0.4 - 0) x 2 = 0.8（取0)

(0.8 - 0) x 2 = 1.6 （取1)

(1.6 - 1) x 2 = 1.2 （取1)

最终结果是0.0010011….（10011循环）
例子2：
0.5 * 2 = 1.0 (取1)

结果为0.1（二进制）
小数二进制转十进制

例子：二进制0.1101

12E-1 + 12E-2; + 1*2E-4 = 0.8125

原码,反码,补码和移码

原码的定义：

①小数原码的定义 
















 

[X]原 =

 X  0≤X ＜1
 1－ X －1 ＜ X ≤ 0

 例如： X=+0.1011 , [X]原= 01011
             X=－0.1011  [X]原= 11011

②整数原码的定义
















 

[X]原 =

 X  0≤X ＜2n
 2n－X －2n ＜ X ≤ 0

 例如： X=+1011 , [X]原= 01011
            X=－1011  [X]原= 11011

原码和反码，补码都是一样的，都是正数本身。

原码的符号位正为0、负为1。原码的加减计算不能将符号位一起纳入加减计算。例如：001(+1)+101(-1)=110（-2）。

很显然结果应该是0。为了解决这种问题，就出现了反码。

反码的定义：

①小数反码的定义 
















 

[X]反 =

 X    0≤X ＜1
 2－2n-1－X   －1 ＜ X ≤ 0

例如：   X=+0.1011      [X]反= 01011
         X=－0.1011     [X]反= 10100

②整数反码的定义
















 

[X]反 =

 X    0≤X ＜2n
 2n+1－1－X    － 2n ＜ X ≤ 0

例如：   X=+1011      [X]反= 01011
         X=－1011     [X]反= 10100

反码是符号位为1,其它位是原码取反。

反码解决了将符号位纳入加减计算的功能，例如：001(+1)+110(-1)=111（0）。

如果空间的大小是3位的，那么反码表示的范围“-3到-0”和“+0到3”。可见-0与+0是多余了一个0。为了解决这个问题即出现了补码。

补码的定义：

①小数补码的定义
















 

[X]补 =

 X   0≤X ＜1
 2＋ X   －1 ≤ X ＜ 0

例如：  X=+0.1011,   [X]补= 01011
        X=－0.1011,  [X]补= 10101

②整数补码的定义
















 

[X]补 =

 X    0≤X ＜2n
 2n+1＋X   － 2n ≤ X ＜ 0

例如：  X=+1011,   [X]补= 01011
        X=－1011,  [X]补= 10101

补码都是一样的，都是正数本身。补码是符号位为1，其它位是原码取反，未位加1。

补码的加法：001(+1)+111(-1)=000(0)

补码设计的目的：

**

⑴使符号位能与有效值部分一起参加运算,从而简化运算规则.

⑵使减法运算转换为加法运算,进一步简化计算机中运算器的线路设计

**

 

移码的定义：移码只用于表示浮点数的阶码，所以只用于整数

移码就是将符号位取反的补码。
