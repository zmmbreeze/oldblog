Prototype库面向对象的实现原理

Prototype库作为一个老牌js的库，其实现面向对象的方法很值得借鉴。先看一段Prototype库的使用代码：

// properties are directly passed to create method
var Person = Class.create({
  initialize: function(name) {
    this.name = name;
  },
  say: function(message) {
    return this.name + ‘: ‘ + message;
  }
});

// when subclassing, specify the class you want to inherit from
var Pirate = Class.create(Person, {
// redefine the speak method
say: function($super, message) {
return $super(message) + ‘, yarr!’;
}
});

var john = new Pirate(‘Long John’);
john.say(‘ahoy matey’);
// -> “Long John: ahoy matey, yarr!”

Javascript的原型链探究

本来想写一篇“如何用JS实现面向对象”，发现自己对prototype原型链还是有很多的不理解的地方。先看一张原型链关系图：

prototype属性

prototype是所有函数都有的一个属性

[js]function Man(){
}
console.log(Man.prototype); //Object{}[/js]

prototype 属性使您有能力向对象添加属性和方法。在每个使用每个对象的属性或方法时，js会按照原型链的顺序查找属性，直到找到。

[js]function employee(name,job,born)
{
this.name=name;
this.job=job;
this.born=born;
}

var bill=new employee("Bill Gates","Engineer",1985);

employee.prototype.salary=null;
bill.salary=20000;

console.log(bill.salary); //20000
[/js]

注意对象是没有prototype属性的，只有函数有。但是一些特有的浏览器（firefox，chrome）通过proto属性暴露了原型prototype，如上图。

new关键字

new用于新建一个对象，例如：

[js]function Man(){
}
var m = new Man();[/js]
下面的代码用js模拟了new操作：
[js]function newObj(Fun,arguments) {
var o = {};
if (Fun && typeof Fun === "function") {
o.proto = Fun.prototype;
Fun.apply(o, arguments);
return o;
}
}[/js]

从代码中可以看出，首先新建一个对象o，然后修改proto指向Fun.prototype，然后以o为上下文(context)执行Fun函数，最后返回o。因为对象的proto设置是在new操作中的，所以导致了以下现象：

[js]function Man(){
}
function Father() {
this.name=""
}
var m = new Man();
Man.prototype = new Father();
console.log(m.name); //undefine
[/js]

至此我们理解了图中的第一层，接下来讲第二、三层。

为何Foo.proto !== Foo.prptotype?

这里Foo函数可以看成是Function函数的对象！按照图中第一层的逻辑：对象的proto指向其函数的prototype属性，Foo的proto应该等于Function.prototype。

那么为什么Function函数和Object函数的proto都等于Function.prototype呢？

注意Function和Object都是函数，而所有的函数都是Function函数的对象（有点绕）！所以同上，它们的proto应该等于Function.prototype。

为什么o1与o2的proto等于Object.prototype呢？

原因与第一层的结构一样，o1,o2是Object函数的对象。

注意：Function.prototype与Foo.prototype同样是对象，它们也是通过Object函数构建的，所以它们的proto也等于Object.prototype。

Object.prototype

Object.prototype值是无法修改，它提供了一些默认的方法。且它的proto等于null！

[js]function Man(){
this.oo = "11";
}
Object.prototype = new Man();
console.log(Object.prototype.oo); //undefine
[/js]

理解Javascript的原型链的重点在于：对象、函数、函数的prototype，函数与函数的prototype同时也是对象。

在JavaScript语言中，所有对象的原型链根节点都是Object.prototype。

instanceof操作符

instanceof是一个二元运算符，如：A instanceof B. 其中，A必须是一个合法的JavaScript对象，B必须是一个合法的JavaScript函数 (function). 判断过程如下：

如果函数B在对象A的原型链 (prototype chain) 中被发现，那么instanceof操作符将返回true，否则返回false.

所以导致了:

[js]Function instanceof Object;//true
Object instanceof Function;//true
Function instanceof Function;//true
Object instanceof Object;//true
[/js]

这样的情况发生！

如何用Javascript实现面向对象编程

自认学javascript一年多了，小致基本类型，大致线程模型该知道的都知道。可是有次面试被问到“如何用Javascript实现面向对象编程？”，居然被问蒙了（因为当时刚出差回来，有休息了一个礼拜，一个多月未碰代码了）。现在冷静的仔细想想，其实挺容易的。

关键的知识在js的prototype。js是prototype-based的编程语言，不是想java那样class-based的编程语言。但是经过一定的变化，js也可以实现OO。

OO的特点：封装、抽象、继承、多态。

封装：在面向对象语言中，封装特性是由类来体现的，我们将现实生活中的一类实体定义成类，其中包括属性和行为（在Java中就是方法），就好像人类，可以具有name,sex,age等属性，同时也具有eat(),sleep()等行为，我们在行为中实现一定的功能，也可操作属性，这是面向对象的封装特性；
抽象：抽象就是将一类实体的共同特性抽象出来，封装在一个抽象类中，所以抽象在面向对象语言是由抽象类来体现的。比如鸟就是一个抽象实体，因为抽象实体并不是一个真正的对象，它的属性还不能完全描述一个对象，所以在语言中体现为抽象类不能实例化；
继承：继承就像是我们现实生活中的父子关系，儿子可以遗传父亲的一些特性，在面向对象语言中，就是一个类可以继承另一个类的一些特性，从而可以代码重用，其实继承体现的是is-a关系，父类同子类在本质上还是一类实体；
多态：多态就是通过传递给父类对象引用不同的子类对象从而表现出不同的行为，多态可为程序提供更好的可扩展性，同样也可以代码重用。

封装在js中很容易实现：

[js]function Man() {
this.name = "";
this.age = 18;
this.work = function() {
//coding();
}
}
var mzhou = new Man();
mzhou.name = "mzhou";
mzhou.age = 23;[/js]

以上代码用Man类封装了人这一现实生活中的一类实体。

抽象在js中不能完美的实现，因为不可能实现一个自己不能实例化的类。但是如果是纯抽象的，那倒是可以（和上面代码类似）。

继承的实现就需要使用到之前所说的prototype了。

[js]function Father() {
this.name = "father";
}
function Son() {
this.age = 18;
}
Son.prototype = new Father();
var son = new Son();
console.log(son.name);//father
son.name = "son";
console.log(son.name);//son
[/js]

如上代码中，Son类继承了Father类的属性。同样方法的继承也同样可以实现。

以下是上面代码的原型链示例图：

因为js是弱类型语言，所以不能实现多态：

[js]
//Shape是Triangle的父类
public static void main(String[] args){
Triangle tri = new Triangle();
System.out.println("Triangle is a type of shape? " + tri.isShape());// 继承
Shape shape = new Triangle();
System.out.println("My shape has " + shape.getSides() + " sides."); // 多态
Rectangle Rec = new Rectangle();
Shape shape2 = Rec;
System.out.println("My shape has " + shape2.getSides(Rec) + " sides."); //重载
}
[/js]

并且也没有必要实现多态，因为它是弱类型的。

override重写与overload重载

重写在之前的例子（继承）中已经演示过了，子类可以重写父类的属性值与方法。

overload重载是很多面向对象语言都有的特性，在js中不能完全实现，但是js可以使用以下的写法达到相同的目的：

[js]function overload(a,b) {
if (b) {
overload_a_b(a,b);
}
else {
overload_a(a);
}
}
function overload_a_b(a,b) {
//处理参数为a和b的情况
}
function overload_a(a) {
//处理参数为a和b的情况
}
[/js]

总结：javascript毕竟不是纯面向对象的语言，所以没有必要用OO的方式来写代码。jQuery就是一个很好的例子！

jQuery的风格

前一篇文章谈到了jQuery的风格。jQuery最颠覆传统编程风格的一点是：它将所有没有返回值的函数返回了调用对象！

$('#id').append("<div>test div</div>")
    .css("background-color","yellow")
    .width(300)
    .height(400);

按照一般的编程风格应该是这样的：

var id = $('#id').append("<div>test div</div>");
id.css("background-color","yellow");//不返回jQuery对象
id.width(300);//不返回jQuery对象
id.height(400);//不返回jQuery对象

对比之下，最显而易见的是字符数很少。所以文件的大小也小。

性能上没有对比过。不过考虑到jQuery烂大街似的使用率就算是性能低，也不会低太多。

另外一个隐性的区别就是：前一种风格更符合一般人的语言习惯，后者则是传统编程人员的思维。这一区别让设计人员也可以轻松上手，因为js和前端是仅仅结合在一起的。

 

缺点：

这种风格不能说有缺点，只能说是容易让一般人造成的性能问题，如下：

$('#id').append("<div>test div</div>")
    .css("background-color","yellow");
$('#id').width(300)
    .height(400);
$('#id').insertBefore("#before");

如上代码，使用了三次$(‘#id’)，这导致了性能大大降低。每次执行都需要重新查找。这种问题编码久了，也就很容易解决这些问题了！

Javascript正则表达式之多行匹配

废话不多说先看例子：

需要匹配字符串（有两行）：

axxxs

axxxs
用于匹配的正则表达式：
/^a([.\n]*)s$/gm

/^a([.\n]*)s$/g

/^a([\s\S]*)s$/gm

/^a(.*)s$/gm

/^a(.\n.)s$/gm
JS代码:

[js]var reg = 正则表达式;
content = content.replace(reg,’$1’);
alert(content);[/js]

结果:

1.axxxs

axxxs

2.axxxs

axxxs

3.xxs

axx

4.xx

xx

5.xxs

axx
准备知识：

  1. .表示查找单个字符，除了换行和行结束符
  2. \n表示换行符
  3. \s表示查找空白字符。
  4. \S表示查找非空白字符。
  5. \s\S表示所有字符即是.和换行和行结束符的合集(\w\W,\d\D也是一样)
  6. n*表示匹配任何包含零个或多个 n 的字符串
  7. n$表示匹配任何结尾为 n 的字符串
  8. ^n表示匹配任何开头为 n 的字符串
  9. g表示执行全局匹配（查找所有匹配而非在找到第一个匹配后停止）
  10. m执行多行匹配，m模式下^和&不仅表示整个表达式的开头与结尾(优先级高)，而且表示单行字符串的开头与结尾(优先级低)。只有在m模式下才会匹配换行符！
  11. [abc]查找方括号之间的任何字符。

  12. $1表示与第一个子表达式相匹配的文本
    分析：

  13. /^a([.\n]*)s$/gm :表示多行查找以a开头以s结尾中间是任意字符的字符串，但是js不支持以[.\n]形式表示任意字符，所以匹配失败

  14. /^a([.\n]*)s$/g :原因与第一个一样，所以匹配失败
  15. /^a([\s\S]*)s$/gm :表示多行查找以a开头以s结尾中间是任意字符的字符串，但是js不支持以[.\n]形式表示任意字符,[\s\S]表示任意字符，多行模式所以支持匹配换行符，而且多行模式下^与&优先表示整个表达式的开头结尾。所以结果为:”xxxs axxx”
  16. /^a(.*)s$/gm：表示多行查找以a开头以s结尾中间是任意字符（除换行符）的字符串，因为多行模式下^和&表示单行字符串的开头与结尾，所以结果为“xxx xxx”;
  17. /^a(.\n.)s$/gm：表示多行查找以a开头以s结尾中间是两行任意字符（除换行符）的字符串，所以结果为”xxxs axxx”;
    附加：
    /^a([\s\S]*)s$/g：因为不是多行模式，所以不匹配换行符，^与$也只表示整个字符串（包括换行符）的起始于结束。所以匹配结果为：”xxxs axxx”

Javascript中的类型详解

http://docs.jquery.com/Types是jquery的文档，主要讲了javascript的type。即使是js的老手看了也会有所帮助，现简要翻译过来！只翻译了通用的部分，jQuery专用的则不考虑。

STRING

 "I'm a String in JavaScript!"
 'So am I!'

Javascript中string是一个包含0或多个字符的不可变对象。对string类型进行typeof操作符操作得到的结果是”string”

Quoting

Javascript中可以用单引号或者是双引号来定义一个字符串。当用双引号来定义一个字符串时其内部可以中使用单引号来表示真正的“单引号”，反之则亦然。
如果使用双引号来定义字符串，那么要想表示真正的双引号则需要使用\来实现转义。

 "You make 'me' sad."
 'Holy "cranking" moses!'
 "<a href=\"home\">Home</a>"

Built-in Methods

string有很多的内建方法来操作字符串，返回的结果是一个新的string或者是其他（如split方法返回一个array）

 "hello".charAt(0) // "h"
 "hello".toUpperCase() // "HELLO"
 "Hello".toLowerCase() // "hello"
 "hello".replace(/e|o/g, "x") // "hxllx"
 "1,2,3".split(",") // ["1", "2", "3"]

Length属性

string的length属性表示字符串的长度。

 "Hello".length // 5
 "".length // 0

Boolean Default

一个空的string，在进行true or fasle判断时，代表false！

NUMBER

在javascript是双精度64位IEEE 754格式的值。和string类型一样，number也是不可变的对象。所有的基于C的一般操作符都能在javascript中使用（+, -, , /, %, =, +=, -=, =, /=, ++, —）。
number的typeof操作值是：

 typeof 12 // "number"
 typeof 3.543 // "number"

Boolean Default

如果number是0，那么就是false。

 !0 // true
 !1 // false
 !-1 // false

由于Number的实现方式是双精度64位IEEE 754格式的值，所以如下的情况不是bug。

0.1 + 0.2 // 0.30000000000000004

Math对象

Javascript中的Math对象提供特殊的数学操作：

 Math.PI // 3.141592653589793
 Math.cos(Math.PI) // -1

Parsing Numbers

parseInt和parseFloat方法被用来把string解析成numbers。如果没有明确指定的话它会进行一些隐式的转换：

 parseInt("123") = 123 (隐式转换为十进制)
 parseInt("010") = 8 (隐式转换为8进制)
 parseInt("0xCAFE") = 51966 (隐式转换为16进制)
 parseInt("010", 10) = 10 (显示转换为10进制)
 parseInt("11", 2) = 3 (显示转换为2进制)
 parseFloat("10.10") = 10.1

Numbers to Strings

当将number附加到string上时结果永远是string。数字相加和附加string的操作符都是“+”，所以要注意。

 "" + 1 + 2; // "12"
 "" + (1 + 2); // "3"
 "" + 0.0000001; // "1e-7"
 parseInt(0.0000001); // 1 (!)

当然你也可以用String方法实现转换：

 String(1) + String(2); //"12"
 String(1 + 2); //"3"

NaN and Infinity

当解析string成number时有可能会导致NaN。这是isNaN方法会帮助你探测这些情况：

 parseInt("hello", 10) // NaN
 isNaN(parseInt("hello", 10)) // true

以0来除一个除一个数会得到无穷大（Infinity）：

1/0; // Infinity

NaN和Infinity都是”number”类型的：

 typeof NaN // "number"
 typeof Infinity // "number"

注意NaN的比较是很奇怪的：

 NaN == NaN // false (!)

但是：

Infinity == Infinity // true

Integer

一个integer是一个普通的number类型。但是每当显示的提到了，表明一个非浮点型的number被期望了。

Float

一个float是一个普通的number类型。但是每当显示的提到了，表明一个浮点型的number被期望了。

BOOLEAN

在javascript中boolean类型可以是true或者false。

 if ( true ) console.log("always!")
 if ( false ) console.log("never!")

OBJECT

在javascript中所有都是object，虽然有些只是类似对象的。创建对象最容易的方法是使用对象符“{}”：

 var x = {};
 var y = {
   name: "Pete",
   age: 15
 };

object的typeof值是“object”：

 typeof {} // "object"

DOT操作符

你可以通过“.”操作符实现对象属性的读写：

 y.name // "Pete"
 y.age // 15
 x.name = y.name + " Pan" // "Pete Pan"
 x.age = y.age + 1 // 16

数组操作符

你也可以使用数组操作符“[]”来实现对象属性的读写：

 var operations = {
   increase: "++",
   decrease: "--"
 }
 var operation = "increase";
 operations[operation] // "++";
 operations["multiply"] = "*"; // "*"

迭代

可以在for-in循环中实现对象的迭代：

var obj = {
  name: "Pete",
  age: 15
};
for(key in obj) {
 alert("key is "+[key]+", value is "+obj[key]);
}

注意的是如果你修改Object.prototype，那么for-in循环也会显示你所添加的属性。

boolean default

一个object，无论它有没有属性，它永远都不是false:

!{} // false

Prototype

所有的对象都有一个prototype属。每当获取一个属性的时候，它也会检查prototype。jQuery使用prototype来为jQuery实体添加方法。

var form = $("#myform");
 form.clearForm; // undefined
 form.fn.clearForm = function() {
   return this.find(":input").each(function() {
     this.value = "";
   }).end();
 };
 form.clearForm() // works for all instances of jQuery objects, because the new method was added to the prototype

Array数组

Javascript中Arrays是一个可变的列表，并提供一些内建方法。你可以通过“[]”来定义数组：

 var x = [];
 var y = [1, 2, 3];

array对象的typeof值为”array”：

 var x = [];
 var y = [1, 2, 3];

你可以通过数组操作符来读写数组元素：

 x[0] = 1;
 y[2] // 3

迭代

你可以通过length属性实现迭代数组：

 for (var i = 0; i < a.length; i++) {
   // Do something with a[i]
 }

当性能需要较强时，可以对length属性进行缓存：

 for (var i = 0, j = a.length; i < j; i++) {
   // Do something with a[i]
 }

另外一种方式，定义变量来存储临时变量。它在数组中包含0或空字符串将不起作用：

 for (var i = 0, item; item = a[i]; i++) {
   // Do something with item
 }

length属性可以被用来添加属性到一个数组。这个push方法一样：

 var x = [];
 x.push(1);
 x[x.length] = 2;
 x // 1, 2

其他内建的方法：

 var x = [0, 3, 1, 2];
 x.reverse() // [2, 1, 3, 0]
 x.join(" – ") // "2 - 1 - 3 - 0"
 x.pop() // [2, 1, 3]
 x.unshift(-1) // [-1, 2, 1, 3]
 x.shift() // [2, 1, 3]
 x.sort() // [1, 2, 3]
 x.splice(1, 2) // [2, 3]

注意：.unshift()方法在IE下不反回数组的长度。

Boolean值

一个array不论它是不是length为0，它永远都不是false：

 ![] // false

Function

一个function可以是有名字的也可以是匿名的。如何函数都可以被赋值到一个变量中，也可以将函数变量传递到函数中。

function named() {}
var handler = function() {}

Arguments变量

在函数中有一个特殊的内部变量“arguments”永远可用，它和数组类似有一个length属性，但是它缺少数组的内建方法。这个伪数组变量代表了函数的参数变量：

function log(x) {
  console.log(typeof x, arguments.length);
}
log(); // "undefined", 0
log(1); // "number", 1
log("1", "2", "3"); // "string", 3

“arguments”对象有一个callee属性，这代表了它所在的function。例如：

 var awesome = function() { return arguments.callee }
 awesome() == awesome // true

Context,Call and Apply

在Javascript中，变量”this”代表了现在的context。默认，”this”对象代表了window对象。在function中this代表的context可以改变，依赖于这个函数如何被调用。
你可以通过call和apply函数实现function的调用和context的修改。

function scope() {
  console.log(this, arguments.length);
}
scope() // window, 0
scope.call("foobar", [1,2]); // "foobar", 1
scope.apply("foobar", [1,2]); // "foobar", 2

Scope作用域

在Javascript中，在function中定义的所有变量都只能在函数作用域中使用。考虑如下例子：

 // global
 var x = 0;
 (function() {
   // private
   var x = 1;
   console.log(x); // 1
 })();
 console.log(x); // 0

它在全局作用域中定义了一个x变量，又在匿名函数中定义了一个局部变量x。内部的x变量值在内部可见。

Closures闭包

每当一个外部作用域的变量在内部作用域被调用了，那么闭包就会被创建了。在如下的例子中，变量counter在create、increment和print函数中是可见的，但是它们不在外部。

function create() {
  var counter = 0;
  return {
    increment: function() {
      counter++;
    },
    print: function() {
      console.log(counter);
    }
  }
}
var c = create();
c.increment();
c.print(); // 1

这种模式能允许你创建一个方法来操作对外不可见的数据。这就是面向对象编程的基本。

代理模式

组合以上的知识可以给你作为一个javascript的编程人员很多可用途径。一种合并方法就是在javascript中实现一个代理模式，实现基本面向对象编程：

(function() {
  // log all calls to setArray
  var proxied = jQuery.fn.setArray;
  jQuery.fn.setArray = function() {
    console.log(this, arguments);
    return proxied.apply(this, arguments);
  };
})();

如上的包装代码的function实现了隐藏proxied变量的功能。它在闭包中存储了setArray方法并重写了它。代理就记录了调用了这个方法的context。使用apply(this,arguments)保证了调用被代理的函数。

Javascript中的“如果...那么...否则...”实现方法探究

在JS中，要实现“如果…那么…否则…”这一逻辑判断有两种方法，“if…else”或者是“? :”操作符。

具体实现：

“if…else”实现方法：

if (Math.round(Math.random()))
   doSomething();
else
   doSomethingElse();

“? :”操作符实现方法：

Math.round(Math.random()) ? doSomething() : doSomethingElse();

区别：

“if…else”实现相比“? :”操作符实现的代码可读性更高一点。

“? :”操作符实现所需要的字符少！这很可能是很多框架使用这种写法的原因之一，节省的字符也会积少成多（对于js来说大小也是很重要的）。

“if…else”实现能够包括更多的代码行，而“? :”操作符只能是一行（不过你可以将这些代码提取成函数，但是这样字符数就不会少了）。

性能的区别：

一直认为这两种写法性能的区别不大，但是今天测试了下，发现还是有些区别的。

测试代码（用到了jquery）：

    function doNothing(message) {
        $(“body”).append(“<div>”+message+”</div>”)
    }
    function testIfElse() {
        if(Math.round(Math.random()))doNothing(“testIfElse true”);else doNothing(“testIfElse false”);
    }
    function testAnd() {
        Math.round(Math.random()) ? doNothing(“testAnd true”) : doNothing(“testAnd false”);
    }
    function test(func,message,times) {
        var start = (new Date).getTime();
        times = times || 100;
        for (var i=0;i<times;i++) {
            func();
        }
        var diff = (new Date).getTime() - start;
        $(“<div>”+message+diff+”</div>”).appendTo(“#result”);
    }
    $(function() {
        test(testIfElse,”If Else take times:”);
        test(testAnd,”And take times:”);
    });

当然如果你需要包括很多代码行时，而这些代码行不能复用，不适合提取成一个函数，那么if…else仍旧是不错的选择！

“如果…那么…”这样的逻辑，除了if…else之外，也可以使用&&操作符实现。这个没有做过性能测试，不过从广大的开源库中频繁使用&&的情况来讲，应该与上面的情况类似。

JavaScript Math 对象详解

Math 对象并不像 Date 和 String 那样是对象的类，因此没有构造函数 Math()，像 Math.sin() 这样的函数只是函数，不是某个对象的方法。您无需创建它，通过把 Math 作为对象使用就可以调用其所有属性和方法。




































E返回算术常量 e，即自然对数的底数（约等于2.718）。
LN2返回 2 的自然对数（约等于0.693）。
LN10返回 10 的自然对数（约等于2.302）。
LOG2E返回以 2 为底的 e 的对数（约等于 1.414）。
LOG10E返回以 10 为底的 e 的对数（约等于0.434）。
PI返回圆周率（约等于3.14159）。
SQRT1_2返回返回 2 的平方根的倒数（约等于 0.707）。
SQRT2返回 2 的平方根（约等于 1.414）。












































































abs(x)返回数的绝对值。
acos(x)返回数的反余弦值。
asin(x)返回数的反正弦值。
atan(x)以介于 -PI/2 与 PI/2 弧度之间的数值来返回 x 的反正切值。
atan2(y,x)返回从 x 轴到点 (x,y) 的角度（介于 -PI/2 与 PI/2 弧度之间）。
ceil(x)对数进行上舍入。

exp(x)返回 e 的指数。
floor(x)对数进行下舍入。

sqrt(x)返回数的平方根。
